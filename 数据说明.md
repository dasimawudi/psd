# 数据说明文档

## 1. 文档目的

本文档说明当前 Abaqus 批量仿真脚本生成的数据格式、字段含义、单位体系以及目录组织方式。

当前数据由以下脚本生成：

- `scripts/generate_lhs.py`
- `scripts/simulate_abaqus.py`
- `scripts/parallel_simulate.py`

输出数据主要用于后续 GNN 训练与数据检查。

---

## 2. 输出目录结构

批量仿真输出目录通常形如：

```text
sim_out_xxx/
├─ case0/
│  ├─ nodes.csv
│  ├─ edges.csv
│  └─ global.json
├─ case1/
│  ├─ nodes.csv
│  ├─ edges.csv
│  └─ global.json
├─ run_info.txt
├─ parallel_summary.txt
├─ _parallel_logs/
└─ _parallel_work/   # 运行中临时目录
```

说明：

- `caseN/` 表示第 `N` 条 DOE 样本对应的一次仿真结果。
- 一个完整成功的 case 必须同时包含：
  - `nodes.csv`
  - `edges.csv`
  - `global.json`
- `run_info.txt` 记录每个 case 的运行状态和耗时。
- `parallel_summary.txt` 记录每个 worker 的整体运行情况。
- `_parallel_logs/` 是并行 worker 的日志目录。
- `_parallel_work/` 是并行运行过程中的工作目录，正常情况下属于中间文件，不建议长期保留。

如果运行时使用了 `--keep-odb`，则 case 目录下还可能包含：

- `tmp/`

其中会保留 `.odb`、`.dat`、`.msg`、`.sta`、`.simdir` 等中间分析文件。

---

## 3. 单位体系

当前脚本采用的单位体系为：

- 长度：`mm`
- 质量：`t`
- 应力：`MPa`
- 加速度：`mm/s^2`
- 频率：`Hz`

其中：

- 输入参数 `Add_mass` 在 DOE 中的单位是 `kg`
- 写入 Abaqus 前会转换为 `t`
- PSD 谱按 `g^2/Hz` 输入
- 参考重力加速度取 `g = 9800 mm/s^2`

---

## 4. case 目录中的 3 个核心文件

### 4.1 `nodes.csv`

`nodes.csv` 为节点级数据，既包含 GNN 节点输入，也包含节点监督标签。

表头如下：

```csv
node_id,x,y,z,bc_mask,RTA,RMises
```

字段说明：

- `node_id`
  - 含义：节点编号
  - 类型：整数
  - 说明：脚本重新按 `0,1,2...` 编号，便于图数据构建

- `x, y, z`
  - 含义：节点三维坐标
  - 类型：浮点数
  - 单位：`mm`
  - 说明：为装配后的整体全局坐标

- `bc_mask`
  - 含义：边界条件掩码
  - 类型：整数
  - 取值：
    - `1`：固定边界节点
    - `0`：非固定节点
  - 说明：当前脚本中，固定节点主要来自耳孔约束区域

- `RTA`
  - 含义：总加速度 RMS 响应
  - 类型：浮点数
  - 单位：`mm/s^2`
  - 说明：对应随机响应分析输出的全场总加速度 RMS

- `RMises`
  - 含义：Mises 应力 RMS 响应
  - 类型：浮点数
  - 单位：`MPa`
  - 说明：节点处由单元节点应力平均得到

#### GNN 对应关系

- 节点输入特征 `X_v`：
  - `[x, y, z, bc_mask]`
- 节点监督标签 `Y_v`：
  - `[RTA, RMises]`

---

### 4.2 `edges.csv`

`edges.csv` 为图的边信息，由实体网格连通关系提取而来。

表头如下：

```csv
src,dst,dx,dy,dz,dist
```

字段说明：

- `src`
  - 含义：起点节点编号
  - 类型：整数

- `dst`
  - 含义：终点节点编号
  - 类型：整数

- `dx, dy, dz`
  - 含义：从 `src` 指向 `dst` 的坐标差
  - 类型：浮点数
  - 单位：`mm`

- `dist`
  - 含义：边长度
  - 类型：浮点数
  - 单位：`mm`

#### GNN 对应关系

- 边输入特征 `X_e`：
  - `[dx, dy, dz, dist]`

---

### 4.3 `global.json`

`global.json` 为 case 级全局信息，包括输入参数、模态频率和 PSD 载荷信息。

当前结构示例如下：

```json
{
  "params": {
    "earpiece_thickness": 4.57,
    "earpiece_RadialDist": 148.69,
    "earpiece_TopWidth": 24.43,
    "earpiece_HoleTopDist": 15.15,
    "earpiece_TopFilletRadius": 10.12,
    "earpiece_BottomFilletRadius": 3.74,
    "plate_radius": 103.19,
    "Add_mass": 0.0323
  },
  "params_list": [...],
  "frequencies": [...],
  "freq_top3": [...],
  "psd_points": [...]
}
```

字段说明：

- `params`
  - 含义：输入参数字典
  - 用途：人工检查最方便
  - 单位：
    - 几何参数：`mm`
    - `Add_mass`：`kg`

- `params_list`
  - 含义：同一组输入参数的列表形式
  - 用途：便于模型直接拼接全局特征
  - 顺序固定为：
    1. `earpiece_thickness`
    2. `earpiece_RadialDist`
    3. `earpiece_TopWidth`
    4. `earpiece_HoleTopDist`
    5. `earpiece_TopFilletRadius`
    6. `earpiece_BottomFilletRadius`
    7. `plate_radius`
    8. `Add_mass`

- `frequencies`
  - 含义：频率步中提取到的模态频率列表
  - 单位：`Hz`
  - 说明：列表中可能出现 `0.0`，通常对应零频或刚体项

- `freq_top3`
  - 含义：前三阶非零固有频率
  - 单位：`Hz`
  - 说明：这是最推荐直接用于全局特征的模态信息

- `psd_points`
  - 含义：随机振动 PSD 谱的定义点
  - 格式：`[PSD值, 虚部, 频率]`
  - 单位：
    - PSD值：`g^2/Hz`
    - 频率：`Hz`
  - 当前脚本中使用的谱点为：
    - `[0.005, 0.0, 20.0]`
    - `[0.12, 0.0, 100.0]`
    - `[0.12, 0.0, 1000.0]`
    - `[0.04, 0.0, 2000.0]`

#### GNN 对应关系

- 全局输入特征 `U`
  - 可由 `params`
  - 或 `params_list + freq_top3 + psd_points`
  - 组合得到

## 5. 输入参数说明

当前模型中写入 `global.json -> params` 的 8 个主要输入参数如下：

- `earpiece_thickness`
  - 耳片厚度
  - 单位：`mm`

- `earpiece_RadialDist`
  - 耳片孔心到圆盘圆心距离
  - 单位：`mm`

- `earpiece_TopWidth`
  - 耳片上底宽度
  - 单位：`mm`

- `earpiece_HoleTopDist`
  - 耳片孔心到上底距离
  - 单位：`mm`

- `earpiece_TopFilletRadius`
  - 耳片顶部圆角半径
  - 单位：`mm`

- `earpiece_BottomFilletRadius`
  - 耳片根部圆角半径
  - 单位：`mm`

- `plate_radius`
  - 圆盘半径
  - 单位：`mm`

- `Add_mass`
  - 圆盘中心附加集中质量
  - 单位：`kg`

## 6. 物理含义说明

### 6.1 `RTA`

`RTA` 表示总加速度 RMS 值。

这里不是某一时刻的瞬时加速度，而是随机响应频域分析得到的统计量。可以理解为：

- 在当前 PSD 随机振动环境下
- 节点总加速度响应的均方根大小

### 6.2 `RMises`

`RMises` 表示 Mises 应力 RMS 值。

同样地，它不是某一时刻的峰值应力，而是随机响应环境下的统计 RMS 应力水平。

### 6.3 `freq_top3`

`freq_top3` 是前三阶非零固有频率，可用于：

- 模态特征检查
- 数据筛选
- 作为全局输入特征参与 GNN 训练

## 8. 如何判断一个 case 是否有效

推荐按以下标准判断：

### 8.1 成功 case

满足以下条件即可认为该 case 可用于训练：

- `caseN/nodes.csv` 存在
- `caseN/edges.csv` 存在
- `caseN/global.json` 存在

### 8.2 失败 case

如果 `run_info.txt` 中出现以下状态，则该 case 不能直接用于训练：

- `EXTRACT_FAIL`
- `SKIP`

常见失败原因包括：

- Abaqus 输入文件处理失败
- ODB 不完整
- 随机响应步没有有效输出帧
- 内存不足或磁盘空间不足

### 8.3 `run_info.txt` 的作用

`run_info.txt` 是逐 case 的运行记录文件，主要用于排查哪些样本成功、哪些样本失败，以及每个样本耗时如何。

典型用途：

- 检查是否存在 `EXTRACT_FAIL`
- 统计本批次成功 case 数量
- 排查异常慢的样本
- 作为断点续跑时的参考

实际字段会随脚本版本略有调整，但通常至少包含以下信息：

- `case_id`
  - case 编号
- `status`
  - 运行状态，例如 `OK`、`EXTRACT_FAIL`、`SKIP`
- `elapsed_sec`
  - 单个 case 的运行时间，单位为秒
- `message`
  - 失败原因或补充说明

### 8.4 `parallel_summary.txt` 的作用

`parallel_summary.txt` 是并行批量运行的汇总文件，主要记录每个 worker 处理了哪些 case、是否异常退出，以及整体总耗时。

这个文件更适合看批处理层面的状态，不用于单个样本训练。

## 9. 训练前推荐筛选规则

建议在正式构建 GNN 数据集前，对每个 `caseN` 做一次快速筛选：

- `caseN/nodes.csv` 存在且文件大小大于 0
- `caseN/edges.csv` 存在且文件大小大于 0
- `caseN/global.json` 存在且可正常解析
- `global.json` 中 `freq_top3` 至少包含 3 个非零频率
- `nodes.csv` 中 `RTA` 和 `RMises` 不应全部为 0
- `run_info.txt` 中对应 case 状态应为成功

如果要做更严格的数据清洗，还可以继续检查：

- 节点数是否低于预期下限
- 边数是否低于预期下限
- 是否存在明显异常值，例如极大 `RTA` 或 `RMises`

---

## 10. 与 GNN 数据集的对应关系

一个 `caseN` 可以直接对应一条图样本：

- 图节点集合 `V`
  - 来自 `nodes.csv`

- 图边集合 `E`
  - 来自 `edges.csv`

- 全局特征 `U`
  - 来自 `global.json`

- 节点监督标签
  - 来自 `nodes.csv` 中的 `RTA` 和 `RMises`

因此可以将当前数据组织为：

- 节点特征：`[x, y, z, bc_mask]`
- 边特征：`[dx, dy, dz, dist]`
- 全局特征：`params / params_list / freq_top3 / psd_points`
- 节点标签：`[RTA, RMises]`

---

## 11. 备注

- 当前输出坐标为装配后的全局坐标，不是单独零件局部坐标。
- 节点数和边数会随几何尺寸、耳片数量和网格划分而变化。
- 如果使用较粗网格，节点数和边数会减少。
- 如果使用 `--keep-odb`，磁盘占用会显著增加。
- 大规模批量计算时，建议定期检查：
  - 磁盘空间
  - `run_info.txt`
  - `_parallel_logs/`
